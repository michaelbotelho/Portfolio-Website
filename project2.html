Primary-Backup Server System

View source code ↗

Tech Stack:

  • Google Remote Procedure Call Framework
  • Python

Key Features:

  • Database Maintains Consistency and Availability
  • Send Write Requests to a Distributed Database

This project was created as a collection of microservices to practice working with distributed systems, similar to Amazon's architecture on a micro scale. The project contains a client application, a primary and backup server representing replicas of a database, and a view server service that manages the two servers. Leveraging gRPC (Google Remote Procedure Call) for communication between services, the system allows a user to issue write requests to the server-side database.

When a client issues a write request, the request is sent to a server where it is propagated to any backup server. Once received by the backup server, the write is applied, and then an acknowledgment is propagated backward eventually reaching the client. By the end of this cycle, all servers will be consistent with the client's log file. To maintain availability in the system, each server sends a heartbeat every 5 seconds to a view server where the heartbeat is logged with a time stamp. The view server notifies of a server/network failure when it fails to receive a heartbeat from any server for 15 seconds.
